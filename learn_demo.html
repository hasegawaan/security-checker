HTTPS の仕組み

HTTP · SSL/TLS · セキュリティヘッダー · 攻撃手法

HTTP vs HTTPS

🔓
HTTP
データが平文で流れる。
途中で誰でも読める。
🔒
HTTPS
データが暗号化される。
途中では読めない。

HTTPS 通信の流れ

1
ブラウザが https:// のサイトにアクセス
2
サーバーが SSL証明書（身分証）を提示
3
ブラウザが証明書の 真偽を検証
4
暗号化された通信が始まる 🔒

セキュリティヘッダー一覧

HSTS
Strict-Transport-Security
HTTPSを強制。HTTPへのダウングレードを防ぐ。
XFO
X-Frame-Options
iframeへの埋め込みを禁止。クリックジャッキング対策。
CSP
Content-Security-Policy
外部スクリプトの読み込みを制限。XSS対策。
XCTO
X-Content-Type-Options
ファイルタイプの自動判定を禁止。
